# Project Rules and Guidelines

## Code Style
1. Use consistent indentation (2 spaces)
2. Write meaningful variable and function names
3. Add comments for complex logic
4. Keep functions small and focused
5. Follow DRY (Don't Repeat Yourself) principles

## Git Practices
1. Write clear commit messages
2. Create feature branches for new work
3. Keep commits atomic and focused
4. Pull and test before pushing
5. Review code before merging

## Documentation
1. Keep README up to date
2. Document API changes
3. Include setup instructions
4. Comment on complex algorithms
5. Update documentation when code changes

## Testing
1. Write unit tests for new features
2. Maintain test coverage
3. Run tests before committing
4. Document test cases
5. Fix failing tests promptly

## Security
1. Never commit sensitive data
2. Use environment variables for secrets
3. Keep dependencies updated
4. Follow security best practices
5. Report security issues immediately

## Code Review
1. Review all pull requests
2. Provide constructive feedback
3. Check for edge cases
4. Verify test coverage
5. Ensure documentation is updated

## Performance
1. Optimize critical paths
2. Monitor resource usage
3. Cache when appropriate
4. Minimize database queries
5. Profile code when needed

## Collaboration
1. Communicate changes clearly
2. Help team members when needed
3. Share knowledge
4. Be respectful in discussions
5. Follow project timeline

Feel free to update these rules as the project evolves.
